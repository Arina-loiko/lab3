## Лабораторная работа №3. Работа с Markdown-разметкой и базовое использование LaTeX в документации проекта
#### Markdown — это облегчённый язык разметки, созданный Джоном Грубером в 2004 году. Его ключевая философия — чтобы текст, отформатированный с помощью Markdown, оставался читаемым и в исходном виде, без необходимости разбираться в сложных тегах, как в HTML. Синтаксис основан на интуитивно понятных символах, которые имитируют то, как люди неформально выделяют элементы в обычном тексте. Например, для создания заголовка используется решётка (#), для жирного начертания — двойные звёздочки (**), а для списка — дефис (-). Файлы Markdown имеют расширение .md или .markdown. По сути, это текст-ориентированный способ написания для веба, который затем легко конвертируется в структурно правильный HTML.
#### Сфера применения Markdown чрезвычайно широка, особенно там, где важны простота, переносимость и сосредоточенность на контенте.
Веб-разработка
Блоги и платформы для написания
Электронные письма и формулы
Статические сайты
Документация
Системы управления версиями
Презентации
#### Markdown нужен разработчикам потому, что он позволяет быстро и без отвлечения оформлять текст прямо в коде или терминале, идеально работает с Git для версионирования документации, является универсальным и читаемым форматом, а также стал стандартом для профессионального общения в среде разработки, улучшая ясность и автоматизацию рабочих процессов.
